# Лабораторная работа 01. Реализация RPC-сервиса с использованием gRPC
## Цель работы
Освоить принципы удаленного вызова процедур (RPC) и их применение в распределенных системах
Изучить основы фреймворка gRPC и языка определения интерфейсов Protocol Buffers (Protobuf)
Научиться определять сервисы и сообщения с помощью Protobuf
Реализовать клиент-серверное приложение на языке Python с использованием gRPC
Получить практические навыки в генерации кода, реализации серверной логики и клиентских вызовов для различных типов RPC.

## Вывод
В ходе выполнения лабораторной работы был успешно разработан и протестирован клиент-сервисный сервис Analytics с использованием технологииgRPC.
В процессе работы были освоены и продемонстрированы следующие ключевые навыки:

#### Определение контракта сервиса.
С помощью языка определения интерфейсов Protocol Buffers(в файле.proto) была создана строгая схема взаимодействия, включающая сервисы, методы и типы сообщений. Это обеспечивает строгую типизацию и независимость от языка реализации.
#### Автоматическая генерация кода.
Использовались инструменты grpcio-tools для автоматической генерации Python-кода из .proto-файла, что значительно упростило и ускорило разработку, создав готовый каркас для клиента и сервера.
#### Реализация различных типов RPC.
Были успешно реализованы два фундаментальных типа gRPC-взаимодействия:
a) UnaryRPC(простой "запрос-ответ") для метода GetCurrentWeather, который идеально подходит для получения мгновенных, атомарных данных.
b) Server StreamingRPC (потоковая передача данных от сервера) для метода StreamEvents(DateRange), что эффективно решает задачу монинга событий.
#### Создание и запуск компонентов.
Были написаны и запущены полноценные серверная (server.py) и клиентская (client.py) части приложения. Сервер был настроен на асинхронную обработку запросов, а клиент продемонстрировал способность корректно обрабатывать поток данных от сервера. Лабораторная работа позволила на практике изучить полный цикл создания микросервиса с помощью gRPC, от проектирования API до его реализации и тестирования. Созданное приложение полностью соответствует поставленному заданию и демонстрирует ключевые преимущества gRPC, такие как производительность, строгая типизация и гибкость в моделях взаимодействия
